# GUI Object 概要

このドキュメントは、MatrixEngine における **GUI Object（GUIオブジェクト）** の全体像を把握するための概要資料です。

ここでは、個々のプロパティや実装方法には踏み込まず、
**「どのような種類の GUI Object があり、どのような役割を担うのか」**を中心に説明します。

詳細な操作手順や設定方法については、各チュートリアルやガイド、ヘルプ資料を参照してください。

---

## GUI Object とは

MatrixEngine では、HMI 画面を構成する部品を **GUI Object** として定義し、
ボタンやリスト、表示部品などをオブジェクト単位で設計・管理します。

GUI Object を中心に設計することで、
- 画面構成の再利用性向上
- デザイナーとエンジニアの役割分担
- UI 構造の整理と保守性向上
を実現します。

---

## GUI Object の主な分類

GUI Object は用途に応じて、いくつかのカテゴリに分けて考えることができます。
以下は代表的な分類と例です。

---

### 1. ボタン・操作系 GUI Object

ユーザーの入力や操作を受け付けるための GUI Object 群です。

**代表的な例：**
- Button Control（単一ボタン）
- Button Group Control（ボタングループ）
- Slider Control（スライダー）
- Tab Control（タブ切り替え）
- Panel Control（操作パネル）

主に、
- コマンド実行
- 状態切り替え
- 値の調整
といった用途で使用されます。

---

### 2. リスト・コンテナ系 GUI Object

複数の項目や情報をまとめて表示・管理するための GUI Object 群です。

**代表的な例：**
- List Control（リスト表示）
- HList Control（横方向リスト）
- Drum Control（ドラム型選択）
- HDrum Control（横型ドラム）

データ一覧や選択 UI を構築する際に利用されます。

---

### 3. 状態・進行表示系 GUI Object

システムや処理の状態をユーザーに伝えるための GUI Object 群です。

**代表的な例：**
- Progress Control（進行状況表示）
- Scroller Control（スクロール位置表示）

処理の進捗や現在位置などを視覚的に表現する用途に使われます。

---

### 4. 共通・汎用機能 GUI Object

特定の表示や操作に限定されず、複数の GUI Object から利用される共通機能群です。

**例：**
- 基本機能（Basic Function）
- イベント管理
- イージング・アニメーション関連
- 共通ビルドオプション

これらは直接画面に表示されるものではなく、
GUI Object の振る舞いや連携を支える役割を持ちます。

---

### 5. 拡張・管理系機能

GUI Object の管理や拡張を目的とした機能群です。

**例：**
- Focus 管理
- アクセシビリティ管理
- ブロードキャストイベント管理
- コントロール管理機能

大規模な HMI 構成や高度な操作性を実現する際に利用されます。

---

## 補足

- 本ドキュメントで紹介している GUI Object 名は、
  代表的な例を示したものであり、すべてを網羅するものではありません。
- 実際に使用可能な GUI Object や機能は、
  使用する MatrixEngine のバージョンや設定によって異なる場合があります。
